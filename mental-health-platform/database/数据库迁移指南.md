# 数据库迁移指南

## 执行顺序

按照以下顺序执行SQL文件：

### 1. 创建数据库（如果还没创建）
```bash
mysql -u root -p < database/01-create-database.sql
```

### 2. 创建所有表
```bash
mysql -u root -p mental_health < database/04-create-tables-final.sql
```

### 3. 初始化基础数据
```bash
mysql -u root -p mental_health < database/05-init-data.sql
```

### 4. 插入测试数据
```bash
mysql -u root -p mental_health < database/07-insert-test-data.sql
```

### 5. 插入敏感词数据
```bash
mysql -u root -p mental_health < database/08-insert-sensitive-words.sql
```

### 6. 创建文章收藏表
```bash
mysql -u root -p mental_health < database/09-create-article-collect-table.sql
```

### 7. 创建通知表
```bash
mysql -u root -p mental_health < database/10-create-notification-table-utf8.sql
```

### 8. 创建树洞点赞和举报表（最新）
```bash
mysql -u root -p mental_health < database/11-create-moment-like-report-tables.sql
```

## 快速执行（一键迁移）

如果你想一次性执行所有迁移，可以使用：

```bash
cd mental-health-platform
mysql -u root -p < database/all-in-one.sql
```

## 常见问题

### 问题1：表已存在
如果提示表已存在，可以先删除再创建：
```sql
DROP TABLE IF EXISTS moment_like;
DROP TABLE IF EXISTS moment_report;
DROP TABLE IF EXISTS moment_comment_like;
```

### 问题2：字符集问题
确保数据库使用UTF-8字符集：
```sql
ALTER DATABASE mental_health CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 问题3：权限问题
确保root用户有足够权限：
```sql
GRANT ALL PRIVILEGES ON mental_health.* TO 'root'@'localhost';
FLUSH PRIVILEGES;
```
